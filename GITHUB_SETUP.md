# BharatMarkets Pro — GitHub Hosting Guide

## What You'll Set Up

```
github.com/Murugkan/bharatmarkets/
├── index.html                          ← Your app (already built)
├── prices.json                         ← Auto-generated by GitHub Action
├── symbols.json                        ← Auto-generated by GitHub Action
├── charts/
│   ├── NIFTY.json                      ← Auto-generated
│   ├── RELIANCE.json                   ← Auto-generated
│   └── SYM.json ...                    ← One per symbol
├── watchlist.txt                       ← You edit this to add symbols
├── fetch_prices.py                     ← Python script (copy below)
└── .github/
    └── workflows/
        └── fetch-prices.yml            ← GitHub Action (copy below)
```

**Live URL:** `https://murugkan.github.io/bharatmarkets`

---

## Step 1 — Enable GitHub Pages

1. Go to your repo: `github.com/Murugkan/bharatmarkets`
2. Click **Settings** → **Pages** (left sidebar)
3. Under **Source** → select **Deploy from a branch**
4. Branch: **main** · Folder: **/ (root)**
5. Click **Save**
6. Wait ~2 min → your site is live at `https://murugkan.github.io/bharatmarkets`

    yf_sym = sym + ".NS" if sym not in ("NIFTY", "BANKNIFTY", "NIFTY50") else (

        "fwdPe":      info.get("forwardPE")         or None,
        "pb":         info.get("priceToBook")       or None,
        "eps":        info.get("trailingEps")       or None,
        "bv":         info.get("bookValue")         or None,
        "roe":        round(float(info.get("returnOnEquity") or 0) * 100, 2) or None,
        "divYield":   info.get("dividendYield")     or 0,
        "w52h":       info.get("fiftyTwoWeekHigh")  or None,
        "w52l":       info.get("fiftyTwoWeekLow")   or None,
        "ma50":       info.get("fiftyDayAverage")   or None,

## Step 4 — Add the GitHub Action

Create `.github/workflows/fetch-prices.yml`:

```yaml
name: Fetch NSE Prices

on:
  schedule:
    # Every 15 min, Mon–Fri, 9:00–15:30 IST (03:30–10:00 UTC)
    - cron: '*/15 3-10 * * 1-5'
  workflow_dispatch:   # Manual trigger button in GitHub UI

jobs:
  fetch:
    runs-on: ubuntu-latest
    permissions:
      contents: write

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.12'

      - name: Install dependencies
        run: pip install yfinance

      - name: Fetch prices & charts
        run: python fetch_prices.py

      - name: Commit & push
        run: |
          git config user.name  "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add prices.json symbols.json charts/
          git diff --staged --quiet || git commit -m "chore: update prices $(date -u +%H:%M)"
          git push
```

## Troubleshooting

| Problem | Fix |
|---|---|
| `prices.json` not loading | Run the Action manually (Step 6) |
| Prices stale | Check Action is enabled in repo Settings → Actions |
| Symbol not found in search | Add to `watchlist.txt`, re-run Action |
| Chart missing for a symbol | Add symbol to `watchlist.txt`, Action writes `charts/SYM.json` |
| Safari CORS error | Ensure app is served from `github.io` — not `localhost` |
| API rate limit (yfinance) | Increase `time.sleep(0.4)` in `fetch_prices.py` |
